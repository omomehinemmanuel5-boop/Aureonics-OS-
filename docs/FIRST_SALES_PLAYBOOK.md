# LEX AUREON — First Sales Playbook

## 1) Offer Definition (What to sell first)

### Product category
**AI Truth Layer as a Service**

### Core promise
“See exactly when model behavior drifts, and prove when governance corrected it.”

### Initial commercial package
- **Name:** Trust Visibility Pilot
- **Duration:** 14–28 days
- **Scope:** single use case, single environment, live runtime observation
- **Outputs:** intervention telemetry, trust report exports, implementation guidance

---

## 2) ICP (Ideal Customer Profile)

Prioritize buyers with immediate risk pressure:
- AI copilots in legal, health, finance, insurance, HR, procurement
- enterprise SaaS vendors in security reviews
- teams running third-party LLMs but lacking intervention observability

Buying roles:
- Head of AI / VP Engineering
- Security / Risk / Compliance leads
- Solutions architects owning enterprise rollout

---

## 3) Demo Narrative (Problem → Proof → Purchase)

### A) Problem statement
“Raw LLM behavior can drift, over-comply, hallucinate, or violate policy without obvious signals.”

### B) Live proof
Run safe / boundary / attack sequence and show:
- RAW output
- governed output
- final decision badge
- intervention rationale

### C) Buying outcome
“Your team gets auditable evidence of AI correction decisions with session-level trust metrics.”

---

## 4) Pricing Strategy (Early Stage)

### Pilot pricing anchor (example)
- **Starter pilot:** $7.5k–$15k / month
- Includes: console access, report exports, weekly advisory readout

### Expansion tiers
1. **Trust Ops**: advanced governance insights + multi-session analytics
2. **Trust API**: API access logs + SIEM-friendly export pipelines
3. **Enterprise**: dedicated environments + SLA + policy customization

---

## 5) Objection Handling

### “Can’t we do this with prompt engineering?”
Prompt engineering does not provide auditable intervention traces and post-hoc trust scoring.

### “We already use moderation APIs.”
Moderation flags risk; LEX demonstrates before/after behavior and intervention impact at runtime.

### “Why now?”
Enterprise AI deals increasingly require demonstrable governance evidence, not claims.

---

### Week 1

### Week 2

- deliver weekly trust reports
- package outcomes into public-safe proof points
